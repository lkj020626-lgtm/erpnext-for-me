# StarMake ERP 管理员手册

## 1. 角色与权限配置

### 1.1 预设角色

| 角色 | 权限范围 |
|------|---------|
| Administrator | 系统管理，全部权限 |
| Boss | 所有业务数据读写 |
| Sales | 客户、销售订单、交货单、销售发票 |
| Purchasing | 供应商、采购订单、采购入库、采购发票 |
| Warehouse | 库存条目、交货单、采购入库、盘点 |
| Production | 生产工单、BOM、领料 |
| Finance | 发票、收付款、会计凭证 |
| Service | 问题工单、客户跟进 |
| Auditor | 所有数据只读 |

### 1.2 创建用户并分配角色

路径：`设置 > 用户 > 新建`

1. 填写邮箱（作为登录名）
2. 设置姓名
3. 在「角色」表格中勾选对应角色
4. 保存

### 1.3 数据权限（按公司/仓库限制）

路径：`设置 > 用户权限 > 新建`

示例：限制仓管员只能看到「成品仓」的数据：
- 用户：warehouse@company.com
- 允许：Warehouse = 成品仓
- 适用于所有相关 DocType

## 2. 工作流配置

路径：`设置 > 工作流 > 新建`

示例：采购订单审批流
- 状态：草稿 → 待审批 → 已批准 → 已提交
- 转换条件：金额 > 5000 需要 Boss 审批

## 3. 自定义字段管理

路径：`设置 > 自定义字段`

StarMake 已添加的自定义字段：
- Item: 规格型号、最低库存、默认供应商、参考采购价
- Customer: 联系人、联系电话、收货地址、备注
- Supplier: 联系人、联系电话、地址、银行信息、备注
- Warehouse: 仓库位置、仓管员

如需添加更多字段，在此界面操作，无需改代码。

## 4. 打印格式维护

路径：`设置 > 打印格式`

StarMake 提供 7 个预设打印格式，如需修改：
1. 找到对应的打印格式
2. 编辑 HTML 内容
3. 保存

打印格式使用 Jinja2 模板语法，可用变量：
- `{{ doc.name }}` - 单据编号
- `{{ doc.items }}` - 明细行
- `{{ frappe.session.user }}` - 当前用户

## 5. 报表维护

### 添加新报表

路径：`设置 > 报表生成器 > 新建`

或在 StarMake 代码中添加 Script Report。

### 修改现有报表筛选条件

编辑对应报表的 JSON 文件中的 filters 数组。

## 6. 数据备份策略

### 自动备份

- 频率：每天凌晨 2:00
- 内容：数据库 + 上传文件 + 私有文件
- 保留：最近 30 天
- 路径：`/home/frappe/frappe-bench/sites/*/private/backups/`

### 手动备份

```bash
# 裸机
bench --site erp.local backup --with-files

# Docker
./scripts/manage.sh backup
```

### 备份到外部存储

建议配置 rsync 或 NAS 挂载：

```bash
# 每天 4:00 同步到 NAS
0 4 * * * rsync -avz /home/frappe/frappe-bench/sites/erp.local/private/backups/ /mnt/nas/erp-backup/
```

## 7. 恢复演练

建议每月执行一次：

```bash
# 1. 创建测试站点
bench new-site test-restore.local --db-root-password ROOT_PASS --admin-password test

# 2. 恢复最新备份
bench --site test-restore.local restore /path/to/latest-backup.sql.gz

# 3. 迁移
bench --site test-restore.local migrate

# 4. 验证
# - 登录检查
# - 商品数量对比
# - 最近订单检查
# - 库存数据检查

# 5. 清理测试站点
bench drop-site test-restore.local --db-root-password ROOT_PASS
```

## 8. 系统巡检

每周检查项：

| 检查项 | 命令 | 正常标准 |
|--------|------|---------|
| 磁盘空间 | `df -h` | 使用率 < 80% |
| 内存使用 | `free -h` | 可用 > 1GB |
| 服务状态 | `supervisorctl status` | 全部 RUNNING |
| 错误日志 | `bench --site erp.local show-pending-jobs` | 无积压 |
| 备份文件 | `ls -lh sites/*/private/backups/` | 有最近备份 |
| 数据库大小 | `bench --site erp.local mariadb -e "SELECT table_schema, ROUND(SUM(data_length+index_length)/1024/1024,2) AS 'Size(MB)' FROM information_schema.tables GROUP BY table_schema;"` | 正常增长 |

## 9. 日志查看

```bash
# 应用日志
tail -f ~/frappe-bench/logs/frappe.log

# Worker 日志
tail -f ~/frappe-bench/logs/worker.log

# Nginx 访问日志
tail -f /var/log/nginx/access.log

# 错误日志
bench --site erp.local show-pending-jobs
```

## 10. 升级注意事项

1. **升级前必须备份**
2. 先在测试环境验证升级
3. 升级后运行 `bench migrate`
4. 检查自定义字段是否正常
5. 检查打印格式是否正常
6. 检查报表是否正常
7. 如有问题，从备份恢复
