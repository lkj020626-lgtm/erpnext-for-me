## 2. 核心扩展机制

### 2.1 hooks.py

所有扩展的入口。主要用到：

- `doc_events` - 在 ERPNext 单据的生命周期中插入自定义逻辑
- `scheduler_events` - 定时任务（如每日备份）
- `fixtures` - 声明需要导出/导入的配置数据
- `app_include_js` - 注入前端 JS
- `after_install` / `after_migrate` - 安装和迁移后执行

### 2.2 Custom Field（自定义字段）

通过 `fixtures/custom_field.json` 定义，安装时自动创建。

添加新字段步骤：
1. 在 JSON 文件中添加记录
2. 运行 `bench --site erp.local migrate`
3. 字段自动出现在对应 DocType 表单中

### 2.3 Property Setter（属性修改器）

通过 `fixtures/property_setter.json` 隐藏或修改原生字段属性。

### 2.4 Print Format（打印格式）

HTML + Jinja2 模板，存放在 `print_templates/formats/`。

### 2.5 Script Report（脚本报表）

每个报表一个目录，包含 `.py`（数据逻辑）和 `.json`（元数据/筛选器）。

## 3. 常见修改方式

### 3.1 添加新的自定义字段

```json
// fixtures/custom_field.json 中添加
{
  "doctype": "Custom Field",
  "dt": "Sales Order",
  "module": "Master Data",
  "fieldname": "custom_delivery_method",
  "fieldtype": "Select",
  "label": "配送方式",
  "options": "\n自提\n送货上门\n物流",
  "insert_after": "delivery_date"
}
```

### 3.2 添加新的 doc_event

```python
# hooks.py 中添加
doc_events = {
    "Sales Order": {
        "on_submit": "starmake.selling.my_new_hook.on_submit",
    },
}
```

```python
# selling/my_new_hook.py
import frappe

def on_submit(doc, method):
    # 你的逻辑
    pass
```

### 3.3 添加新报表

1. 创建目录 `starmake/reports/my_report/`
2. 创建 `my_report.py`（实现 `execute(filters)` 函数）
3. 创建 `my_report.json`（定义筛选器）
4. 创建 `__init__.py`

### 3.4 添加新打印模板

1. 在 `print_templates/formats/` 创建 HTML 文件
2. 在 `fixtures/print_format.json` 注册
3. 在 `install.py` 的 `PRINT_FORMAT_MAP` 中添加映射

### 3.5 添加新的 API 端点

```python
# 在任意模块文件中
import frappe

@frappe.whitelist()
def my_api(param1, param2):
    # 你的逻辑
    return result
```

前端调用：
```javascript
frappe.call({
    method: "starmake.module.file.my_api",
    args: { param1: "value1", param2: "value2" },
    callback: function(r) { console.log(r.message); }
});
```

## 4. 开发环境搭建

```bash
# 1. 初始化 bench
bench init frappe-bench --frappe-branch version-15
cd frappe-bench

# 2. 安装 ERPNext
bench get-app erpnext --branch version-15

# 3. 安装 StarMake（开发模式）
bench get-app /path/to/starmake

# 4. 创建开发站点
bench new-site dev.local --db-root-password root --admin-password admin
bench --site dev.local install-app erpnext
bench --site dev.local install-app starmake

# 5. 启动开发服务器
bench start
```

修改代码后：
```bash
# Python 代码修改后自动重载（bench start 模式下）
# JS 代码修改后需要：
bench build --app starmake

# 添加/修改 fixtures 后：
bench --site dev.local migrate
```

## 5. 升级兼容策略

1. **不修改 ERPNext 核心代码** - 所有定制通过 hooks 和 fixtures
2. **Custom Field 使用 `custom_` 前缀** - 避免与未来 ERPNext 字段冲突
3. **锁定 ERPNext 版本** - 项目期间使用 version-15，不随意升级
4. **升级前测试** - 在测试环境先验证兼容性
5. **保持 fixtures 幂等** - install.py 中的导入逻辑支持重复执行

## 6. 代码规范

- Python: PEP 8, max line length 120
- JS: ESLint with Frappe config
- 文件命名: snake_case
- 函数命名: snake_case
- 类命名: PascalCase
- 所有用户可见文本使用 `_()` 包裹（支持翻译）
- SQL 使用参数化查询（防注入）
