## 2. 方案一：裸机部署（Debian 13）

### 2.1 系统准备

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装基础依赖
sudo apt install -y \
    git python3 python3-dev python3-pip python3-venv \
    nodejs npm \
    mariadb-server mariadb-client \
    redis-server \
    curl wget sudo \
    xvfb libfontconfig wkhtmltopdf \
    fonts-wqy-zenhei fonts-wqy-microhei fonts-noto-cjk \
    nginx supervisor \
    cron logrotate
```

### 2.2 安装 Node.js 18+

```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
node --version  # 确认 >= 18
npm install -g yarn
```

### 2.3 配置 MariaDB

```bash
sudo mysql_secure_installation
# 设置 root 密码，移除匿名用户，禁止远程 root 登录

sudo nano /etc/mysql/mariadb.conf.d/50-server.cnf
```

在 `[mysqld]` 段添加：

```ini
[mysqld]
character-set-client-handshake = FALSE
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci
innodb_buffer_pool_size = 256M
innodb_log_file_size = 64M
max_allowed_packet = 256M

[mysql]
default-character-set = utf8mb4
```

```bash
sudo systemctl restart mariadb
```

### 2.4 创建 Frappe 用户

```bash
sudo adduser --disabled-password --gecos "" frappe
sudo usermod -aG sudo frappe
echo "frappe ALL=(ALL) NOPASSWD:ALL" | sudo tee /etc/sudoers.d/frappe

sudo su - frappe
```

### 2.5 安装 Bench

```bash
# 以 frappe 用户执行
pip3 install frappe-bench

# 初始化 bench
bench init frappe-bench --frappe-branch version-15
cd frappe-bench
```

### 2.6 安装 ERPNext

```bash
bench get-app erpnext --branch version-15
```

### 2.7 安装 StarMake

```bash
# 方式一：从 GitHub 安装
bench get-app https://github.com/lkj020626-lgtm/erpnext-for-me.git

# 方式二：从本地路径安装
bench get-app /path/to/starmake
```

### 2.8 创建站点

```bash
bench new-site erp.local \
    --db-root-password YOUR_MYSQL_ROOT_PASSWORD \
    --admin-password admin123 \
    --db-name starmake_db

bench --site erp.local install-app erpnext
bench --site erp.local install-app starmake
bench --site erp.local migrate

echo "erp.local" > sites/currentsite.txt
```

### 2.9 启动开发模式（测试用）

```bash
bench start
# 访问 http://服务器IP:8000
```

### 2.10 配置生产模式

```bash
sudo bench setup production frappe
sudo bench setup nginx
sudo supervisorctl reload
sudo systemctl restart nginx
# 访问 http://服务器IP
```

### 2.11 配置自动备份

```bash
# 添加 crontab
crontab -e
```

添加以下行：

```
0 2 * * * cd /home/frappe/frappe-bench && bench --site erp.local backup --with-files >> /var/log/frappe-backup.log 2>&1
0 3 * * * find /home/frappe/frappe-bench/sites/erp.local/private/backups -mtime +30 -delete
```

## 3. 方案二：Docker 部署（Debian 13）

### 3.1 安装 Docker

```bash
sudo apt update
sudo apt install -y ca-certificates curl gnupg

sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

sudo systemctl enable --now docker
sudo usermod -aG docker $USER
```

### 3.2 部署 StarMake

```bash
# 克隆仓库
cd /opt
sudo git clone https://github.com/lkj020626-lgtm/erpnext-for-me.git starmake
cd starmake/deployment

# 配置环境变量
cp .env.example .env
nano .env  # 修改密码和站点名

# 首次初始化
chmod +x scripts/*.sh
bash scripts/init.sh
```

### 3.3 日常管理

```bash
cd /opt/starmake/deployment

# 查看状态
./scripts/manage.sh status

# 查看日志
./scripts/manage.sh logs

# 重启
./scripts/manage.sh restart

# 手动备份
./scripts/manage.sh backup

# 更新应用
./scripts/manage.sh update
```

### 3.4 Docker 自动备份

Docker 方案中，scheduler 容器会自动执行每日备份。备份文件存放在 `.env` 中配置的 `BACKUP_PATH` 目录。

如需额外备份到外部存储：

```bash
# 挂载 NAS
sudo mount -t cifs //nas-ip/share /mnt/nas -o username=user,password=pass

# 修改 .env
BACKUP_PATH=/mnt/nas/erp-backup

# 重启
./scripts/manage.sh restart
```

## 5. 生产环境配置

### 5.1 域名配置（裸机）

```bash
bench setup add-domain your-domain.com --site erp.local
sudo bench setup nginx
sudo systemctl restart nginx
```

### 5.2 HTTPS 配置

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

### 5.3 防火墙

```bash
sudo apt install -y ufw
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```

## 6. 备份与恢复

### 6.1 手动备份

```bash
# 裸机
bench --site erp.local backup --with-files

# Docker
./scripts/manage.sh backup
```

### 6.2 查看备份

```bash
# 裸机
ls -lh ~/frappe-bench/sites/erp.local/private/backups/

# Docker
./scripts/manage.sh backup-list
```

### 6.3 恢复到当前站点

```bash
# 裸机
bench --site erp.local restore /path/to/database.sql.gz
bench --site erp.local migrate
bench --site erp.local clear-cache

# Docker
./scripts/manage.sh restore <备份目录名>
```

### 6.4 恢复到新站点（测试恢复）

```bash
bench new-site test.local --db-root-password ROOT_PASS --admin-password test123
bench --site test.local restore /path/to/database.sql.gz
bench --site test.local migrate
bench --site test.local install-app starmake
```

### 6.5 备份保留策略

- 自动保留最近 30 天备份
- 超期自动清理
- 建议每月手动拷贝一份到外部存储

## 7. 升级流程

### 7.1 裸机升级

```bash
cd ~/frappe-bench

# 先备份
bench --site erp.local backup --with-files

# 更新代码
bench update --pull

# 如果只更新 StarMake
cd apps/starmake && git pull && cd ../..
bench --site erp.local migrate
bench build
sudo supervisorctl restart all
```

### 7.2 Docker 升级

```bash
cd /opt/starmake/deployment

# 先备份
./scripts/manage.sh backup

# 拉取最新代码
cd /opt/starmake && git pull

# 重新构建并部署
cd deployment
./scripts/manage.sh update
```

## 8. 常见故障处理

| 问题 | 排查方法 | 解决方案 |
|------|---------|---------|
| 无法访问网页 | `systemctl status nginx` | `sudo systemctl restart nginx` |
| 502 Bad Gateway | `supervisorctl status` | `sudo supervisorctl restart all` |
| 数据库连接失败 | `systemctl status mariadb` | `sudo systemctl restart mariadb` |
| Redis 连接失败 | `systemctl status redis` | `sudo systemctl restart redis-server` |
| 打印乱码 | 检查字体 | `sudo apt install fonts-wqy-zenhei` |
| 磁盘满 | `df -h` | 清理旧备份或扩容 |
| 内存不足 | `free -h` | 重启服务或增加 swap |

### 裸机完整重启

```bash
sudo systemctl restart mariadb
sudo systemctl restart redis-server
sudo supervisorctl restart all
sudo systemctl restart nginx
```

### Docker 完整重启

```bash
cd /opt/starmake/deployment
docker compose down
docker compose up -d
```

## 9. 从测试迁移到生产

```bash
# 1. 在测试环境备份
bench --site test.local backup --with-files

# 2. 将备份文件拷贝到生产服务器
scp /path/to/backup/* user@prod-server:/tmp/

# 3. 在生产环境恢复
bench --site erp.local restore /tmp/database.sql.gz \
    --with-public-files /tmp/files.tar.gz \
    --with-private-files /tmp/private-files.tar.gz

# 4. 迁移
bench --site erp.local migrate
bench --site erp.local clear-cache

# 5. 验证
bench --site erp.local doctor
```

## 10. 无法翻墙 / 国内网络部署方案

如果服务器无法访问 GitHub、Docker Hub 等境外服务，有两种方案：

### 方案 A：离线部署（完全无网络）

在你本机（能翻墙）打包好所有东西，用 U 盘或内网传输到服务器。

**步骤 1：在本机打包**

```bash
cd /path/to/starmake/deployment/offline
bash pack-offline-bundle.sh
# 产出：starmake-offline-bundle.tar.gz（约 2-3 GB）
```

**步骤 2：拷贝到服务器**

```bash
# U 盘、SCP、内网共享均可
scp starmake-offline-bundle.tar.gz user@server:/tmp/
```

**步骤 3：在服务器安装**

```bash
cd /tmp
tar -xzf starmake-offline-bundle.tar.gz
cd starmake-offline-bundle
sudo bash install-offline.sh
```

> 注意：服务器上需要预装 Docker。如果连 Docker 都无法在线安装，
> 可以从 https://download.docker.com/linux/debian/dists/ 下载 deb 包离线安装。

---

### 方案 B：使用国内镜像源（有国内网络）

服务器能访问国内网站但不能翻墙时，配置镜像加速即可。

**一键配置所有镜像源：**

```bash
cd /path/to/starmake/deployment/offline
sudo bash setup-china-mirrors.sh
```

该脚本会配置：
- APT 源 → 阿里云
- Docker 镜像加速 → 多个国内源
- pip 源 → 清华大学
- npm 源 → npmmirror（淘宝）
- Docker CE 安装源 → 阿里云

**配置完成后，正常执行部署即可：**

```bash
# Docker 部署
cd /opt/starmake/deployment
bash scripts/init.sh

# 或裸机部署（国内网络一键脚本）
sudo bash deployment/offline/deploy-china-baremetal.sh
```

---

### 方案 C：本机构建 Docker 镜像后导出

如果你想用 Docker 部署但服务器拉不到镜像：

```bash
# 1. 在本机构建完整镜像
cd /path/to/starmake/deployment
docker compose build

# 2. 导出镜像
docker save starmake-app:latest | gzip > starmake-app.tar.gz
docker save mariadb:10.11 | gzip > mariadb.tar.gz
docker save redis:7-alpine | gzip > redis.tar.gz
docker save nginx:alpine | gzip > nginx.tar.gz

# 3. 拷贝到服务器后导入
docker load -i starmake-app.tar.gz
docker load -i mariadb.tar.gz
docker load -i redis.tar.gz
docker load -i nginx.tar.gz

# 4. 启动
cd /opt/starmake/deployment
docker compose up -d
```

---

### 各方案对比

| 方案 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| A 离线部署 | 完全无网络 | 不依赖任何网络 | 包体大，更新麻烦 |
| B 国内镜像 | 有国内网络 | 最简单，和正常部署一样 | 镜像源偶尔不稳定 |
| C 本机构建导出 | 有内网传输能力 | 镜像预构建好，部署快 | 需要本机有 Docker |
