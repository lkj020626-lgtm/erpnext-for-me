## 二、日常操作

### 2.1 开机自启

系统已配置 Docker 自动重启策略，服务器开机后 ERP 会自动启动，无需手动操作。

如果发现服务未启动：
```bash
cd /opt/starmake
./scripts/manage.sh start
```

### 2.2 查看系统状态

```bash
cd /opt/starmake
./scripts/manage.sh status
```

正常状态下所有服务应显示 `Up`。

### 2.3 重启系统

如果系统卡顿或异常：
```bash
cd /opt/starmake
./scripts/manage.sh restart
```

### 2.4 查看日志

系统出错时查看日志：
```bash
cd /opt/starmake
./scripts/manage.sh logs
```

按 `Ctrl+C` 退出日志查看。

## 三、数据备份

### 3.1 自动备份

系统每天凌晨自动执行备份，备份文件保存在 `/opt/starmake/backups/` 目录。
自动保留最近 30 天的备份，超期自动清理。

### 3.2 手动备份

重要操作前建议手动备份：
```bash
cd /opt/starmake
./scripts/manage.sh backup
```

### 3.3 查看备份

```bash
cd /opt/starmake
./scripts/manage.sh backup-list
```

### 3.4 备份到移动硬盘/NAS

将移动硬盘或 NAS 挂载到服务器后，修改 `.env` 文件中的 `BACKUP_PATH`：
```
BACKUP_PATH=/mnt/nas/erp-backup
```
然后重启服务：`./scripts/manage.sh restart`

## 四、数据恢复

**注意：恢复操作会覆盖当前所有数据，请确认后再执行！**

```bash
cd /opt/starmake
./scripts/manage.sh backup-list    # 先查看可用备份
./scripts/manage.sh restore 备份目录名
```

系统会要求输入 `yes` 确认。

## 五、常见问题

### Q: 浏览器打不开系统
1. 检查服务器是否开机
2. 运行 `./scripts/manage.sh status` 查看服务状态
3. 如果服务未启动，运行 `./scripts/manage.sh start`
4. 检查防火墙是否放行 80 端口

### Q: 系统很慢
1. 运行 `./scripts/manage.sh restart` 重启
2. 检查服务器磁盘空间：`df -h`
3. 检查内存使用：`free -h`

### Q: 忘记管理员密码
联系技术支持，或由有服务器权限的人执行：
```bash
cd /opt/starmake
./scripts/manage.sh shell
bench --site erp.local set-admin-password 新密码
```

### Q: 打印出来是乱码
系统已预装中文字体，如果仍有问题请联系技术支持。
